「なんとかなってしまった」日本の構造的問題

核心：Excel + 属人化 + 「動いているから触るな」文化

日本の製造業では商品情報管理の課題が10〜20年前から存在していたが、「なんとか回っていた」ため放置されてきた。以下が未成熟の真因：

#原因実態
1 Excel地獄 商品マスタがExcel管理。部署ごとに別バージョンが存在。コピペで不整合が日常的に発生。
→「共有ドライブのExcelを更新してください」が業務フロー
2 属人化 「あの人しか知らない」情報が大量。ベテラン社員の退職で情報が消失。
→ 商品情報がローカルPCに保存され、一貫性が保てない
3 縦割り組織 設計→製造→営業→マーケの各部門で別システム・別フォーマット。
→ カタログ用・Web用・EC用で同じ情報を3回手入力
4 「動いているから触るな」 年数回のカタログ更新で回っていたため、投資の緊急性が低かった。
→ 経営層の理解が得られず、予算がつかない
5 PIMの認知不足 欧米では10年前から普及。日本ではPIMという言葉すら知らない企業が62%。
→ ガートナーが解説記事を出すレベルでまだ「入門段階」
6 「作る」文化 vs「管理する」文化 日本の製造業は「ものづくり」には投資するが「情報づくり」には投資しない。
→ 製品開発に数年かけるのに、その情報管理は後回し
しかし、もう「なんとかなる」時代は終わった

EC化率の上昇（BtoB含む）、グローバル展開の加速、そしてDPP規制の施行により、Excel管理では物理的に対応不可能な時代に突入。これが2025-2027年の「PIM導入ラッシュ」の引き金となる。

出典: Contentserv 意識・実態調査, エムタメ 製造業PIM, ガートナー PIM解説

3. ゲームチェンジャー：EU Digital Product Passport (DPP)

DPPとは何か？（一言で）

EU域内で販売する全製品に「デジタルの履歴書」をQRコード付きで義務化する規制。
原材料の産地、CO2排出量、リサイクル可能性、性能データなど100以上のデータ属性を電子的に記録・公開する必要がある。
日本企業でもEU向けに輸出するなら対応必須。

DPP施行タイムライン

2026年7月

EU中央デジタルレジストリ設立
全DPPデータの集中管理基盤が稼働開始

2027年2月

バッテリーパスポート義務化
EV・産業用バッテリー（2kWh超）にQRコード付きDPP必須
→ パナソニック、村田製作所、TDK、日本電産 等に直接影響

2027年内

繊維・鉄鋼カテゴリ
アパレル・履物・鉄鋼製品にDPP適用開始
→ 東レ、帝人、旭化成、日本製鉄、JFE 等に影響

2028年〜

電子機器・家電
消費者向け電子機器にDPP適用拡大
→ ソニー、パナソニック、シャープ、京セラ 等に影響

2030年

建材・化学品・家具
残りのカテゴリに全面適用。約30カテゴリが対象
→ ほぼ全ての製造業に波及

DPPで求められるデータ（バッテリーの例）

カテゴリ具体的なデータ項目
原材料コバルト・リチウム・ニッケルの産地、紛争鉱物の地理的起源
CO2ライフサイクル段階別のカーボンフットプリント（採掘→製造→輸送）
リサイクル再生材含有率、回収率、リサイクル可能性の数値
性能容量、耐久性、効率、State-of-Health メトリクス
人権サプライチェーンのデューデリジェンス（人権・環境基準の遵守）
なぜDPPがPIMの「ゲームチェンジャー」なのか

DPP対応 = 製品情報のデジタル一元管理が法律で義務化される、ということ。
これまで「あると便利」だったPIMが、「ないと売れない」に変わる。
典型的な自動車・電子機器メーカーはサプライヤー500〜5,000社、30〜50ヵ国にまたがるサプライチェーンからデータを集約する必要があり、業界アナリストはデータ基盤構築に12〜18ヶ月が必要と試算。
→ つまり2027年2月に間に合わせるには、2025年中に着手しなければ手遅れ。

出典: EY Japan DPP解説, Centric Software DPP, EU Battery Passport Regulation, Informatica DPP Guide

6. AI × PIM — ゲームが変わった

PIMは「データベース」から「AI基盤」に進化している

2025年を境に、PIMは単なる商品情報の保管場所から、AIが商品情報を理解・生成・配信する中核インフラに変わった。
この変化を象徴するのがAkeneoのMCP（Model Context Protocol）対応。

Akeneo MCP — 何が革命的なのか

Claude / GPT
AIモデル
⇄
MCP Server
ガバナンス層
⇄
Akeneo PIM
商品データ
  • 2025年秋: MCP Server を初リリース。AIモデルからPIMに直接アクセスし、自然言語で商品検索・属性更新・コンプライアンスチェックが可能に
  • 2026年冬: MCP Server 拡張版。AIが製品データを扱う際の「ガバナンス層」として機能。製品モデル・エンリッチメントルール・データ関係性を保持し、AIのハルシネーション（嘘）を防止
  • Data Architect Agent: Akeneo初のAIエージェント。PIMのデータモデル設計を自動化。導入期間を数ヶ月→数日に短縮

あかがねにとっての意味

AI × PIM の流れは脅威でもあり、最大のチャンスでもある。

脅威: Akeneoの Data Architect Agent が「PIM導入コンサルの自動化」を始めている。あかがねの強みである「人が伴走する導入支援」の価値が薄れるリスク。

チャンス: AIはデータの「構造化」を自動化できるが、製造業固有の業務知識（カタログ制作のノウハウ、印刷入稿の暗黙知、多言語の文化的ニュアンス）はAIだけでは代替できない。
あかがねがAI機能を自社PIMに統合すれば、「AI × 60年の製造業ノウハウ」という誰にも真似できないポジションを取れる。

8. 2027年以降の仮説 — こうなる

2027年（転換点）
  • DPPバッテリー規制施行 → バッテリー/自動車メーカーが一斉にPIM導入ラッシュ
  • 繊維・鉄鋼もDPP対応開始 → サプライチェーン全体に波及
  • 「PIM入れてますか？」が取引条件の一部になり始める
  • 日本のPIM導入率: 13% → 推定20-25%に急伸
  • Akeneo/Centric PXMの日本語パートナーが急増。国内SIerが参入ラッシュ
2028年（普及期）
  • 電子機器・家電にDPP拡大 → ソニー、パナソニック等が本格対応
  • AI × PIM が標準化。生成AIなしのPIMは「旧式」扱いに
  • MCP対応が業界標準に。Akeneo以外のPIMベンダーも追従
  • Tier2/Tier3 サプライヤーにもDPP対応要求が波及 → 中小製造業のPIM需要が爆発
  • 日本のPIM導入率: 推定30-35%
2029年（成熟期入口）
  • PIM未導入企業は「デジタル後進企業」としてブランド毀損リスク
  • PIMデータとAIエージェントの統合が当たり前に。営業がAIに「この顧客に合う製品は？」と聞くとPIMから最適解を返す
  • DPPデータがサーキュラーエコノミーの基盤に。リサイクル業者がDPPデータで素材を自動仕分け
  • コンサル価値の変化: 「PIM導入支援」→「PIM × AI × DPP の統合運用支援」に移行
2030年〜（新常態）
  • DPP全面適用（約30カテゴリ）。事実上の全製造業が対象に
  • PIM導入率: 推定50-60%。未導入はもはや例外
  • PIMは「あると便利」から「水道や電気と同じインフラ」になる
  • 競争の軸は「PIMを入れているか」から「PIMをどう活用しているか」に移行
  • あかがねのような「運用伴走型」の価値が再評価される（ツール導入だけでは差がつかないため）
あかがねにとっての「2027年問題」

2027年のDPP施行でPIM市場は「選択」から「必須」に変わる。
しかしこの波に乗れるかどうかは「2025-2026年に何をしたか」で決まる。

大手SIer（DNP/SCSK）は既にContentserv/Centric PXMの販売体制を整えている。
Akeneoは日本パートナーを探している段階。

あかがねが今やるべきこと:
① das-solutionのSEOを修正し「PIM 製造業」で検索上位を取る
② DPP対応コンテンツを大量発信し、2027年の相談需要を先取りする
③ 自社PIMにAI機能を統合し「AI × 60年のノウハウ」を差別化にする

この3つを2026年中に実行しなければ、2027年の波は大手に持っていかれる。
